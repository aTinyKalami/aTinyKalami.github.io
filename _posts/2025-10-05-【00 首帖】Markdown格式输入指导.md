Markdown 是一种轻量级标记语言，它的核心思想是用简单易读、易写的纯文本格式来编写文档，然后可自动转换成带有有效格式的 HTML 页面。

## 1. 文档命名要求

按照Markdown格式撰写的Github所有博客文章都要存放在_posts 文件夹中。单击它，然后单击 创建文件按钮，即可创建一个新的文件。

系统要求文章按照下面规则命名：`年年年年-月月-日日-文件名.md`。

即文件名命名的格式：year-month-day-name.md，其中 year 是四位数，month 和 day 是两位数。“name” 可以自定义，扩展名 md 则用于 Markdown。

## 2. Markdown内容输入格式

使用Markdown格式包括文章标题、文本样式、代码块、超链接等基础语法：

### 2.1 标题

使用 `#` 字符来定义标题级别。提示： 为了美观，建议在 `#` 和标题文字之间加一个空格。

第一行内容以一个#号开始，然后接一个空格，然后是标题文字。这样就可以在markdown格式中生成一级标题。

使用#符号数量对应标题级别，最多支持六级标题（即#为一级标题，##为二级，###为三级，以此类推）。

例如：
# 一级标题  
## 二级标题  
### 三级标题  

GitHub会自动根据标题生成目录，点击可快速跳转。 

### 2.2 强调（粗体与斜体）

使用 `_`和`*` 字符来包裹文本，实现加粗和斜体效果。两个字符效果一样，使用数量不同效果不同。使用1个为斜体。2个为加粗体，3个为粗斜体。

斜体：前后各使用1个字符。

语法： `*斜体文本*` 或 `_斜体文本_`;
效果： *斜体文本*
    
粗体：前后各使用2个字符。

语法： `**粗体文本**`或 `__粗体文本__`;
效果： **粗体文本**
    
粗斜体：前后各使用3个字符。

语法： `***粗斜体文本***` 或 `___粗斜体文本___`;
效果： ***粗斜体文本***

### 2.3 链接与图片

1. 链接-行内式

使用 [] 和 () 字符。用于文中随时带入超链接，便于关联阅读。

语法：` [链接文本]``(链接地址 +"可选的标题")`

效果： [链接文本](链接地址 "可选的标题")

如：` [访问谷歌]``(https://www.google.com "搜索引擎")`

例子： [访问谷歌](https://www.google.com "搜索引擎")

效果： 鼠标悬停显示搜索引擎，点击则访问谷歌URL链接

2. 链接-参考式

同样使用 [] 和 () 字符。用于文中多处使用同一个超链接。 在文章结尾统一给出。

这是一个[参考式链接] [1]的例子。

然后在文档任意位置（通常是末尾）定义：

[1]:（https://example.com "可选标题"）

如文章中文本：这里需要[访问谷歌][1]。

文档末尾定义：

[1]：https://www.google.com"搜索引擎"

(此处未验证，需要在文章中确认）

3. 图片

语法与链接几乎完全相同，只是在最前面加一个 !。

语法：` ![图片替代文本](图片链接地址 "可选的标题") `

例子： ![卡拉米](https://bkimg.cdn.bcebos.com/pic/a686c9177f3e6709c93df7f3269f883df8dcd100ff3d?x-bce-process=image/format,f_auto/quality,Q_70/resize,m_lfit,limit_1,w_536)
