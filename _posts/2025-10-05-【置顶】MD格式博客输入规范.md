# MD（MarkDown）格式博客文档输入规则
## 【1】文档命名

系统要求博客文章命名按照下面规则命名：

`年年年年-月月-日日-文件名.md`

Where `YEAR` is a four-digit number, `MONTH` and `DAY` are both two-digit numbers, and `filename` is whatever file name you choose, to remind yourself what this post is about. `.md` is the file extension for markdown files.

## 【2】内容输入

在GitHub博客使用Markdown格式时，主要需掌握标题、文本样式、代码块、超链接等基础语法。以下是常见用法：
### 标题设置

第一行内容以一个#号开始，然后接一个空格，然后是标题文字。这样可以在markdown格式中生成一级标题。

使用#符号表示标题级别，最多支持六级标题（#为一级标题，##为二级，###为三级，以此类推）。

例如：

textCopy Code
# 一级标题  
## 二级标题  
### 三级标题  

GitHub会自动根据标题生成目录，点击可快速跳转。 

### 文本样式

    ‌加粗‌：使用&zwnj;**或__（如**&zwnj;重要内容**） ‌
    斜体：使用*或_（如*需注意*） ‌
    删除线：使用\~\~（如\~\~错误信息\~\~） ‌
    全部加粗和斜体：使用&zwnj;***（如***&zwnj;全部文本***） ‌

### 代码与超链接

    ‌代码块‌：使用三反引号包裹（如git status） ‌
    ‌超链接‌：方括号内写文字，圆括号内填URL（如[GitHub](https://github.com)） ‌

### 引用

使用>符号标注引用内容，相邻行省略>时自动合并为引用段落。 ‌

### 特殊符号
    ‌下划线‌：使用<ins>（如<ins>下划线文本<ins>） ‌
    ‌目录生成‌：需手动添加:ml-search-more[toc]{text="Markdown toc语法"}标签（GitHub暂未内置此功能）
