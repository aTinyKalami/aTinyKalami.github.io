AI - 具身智能

十二亚晖 2026-04-07 09:00:00
Categories： AI Tags： AI 技术 具身智能

1 VLM、VLN和VLA各是什么？

VLN (Vision-Language Navigation, 视觉 - 语言 - 导航) 与 VLA (Vision-Language-Action, 视觉 - 语言 - 动作) 是具身智能的两大核心模型。
两者的核心区别在于任务目标与能力范围：VLN 专注解决 “去哪里” 的空间移动问题，而 VLA 则解决 “做什么” 的物理交互问题。

而VLM（Vision-Language Model）是指视觉语言模型，比如 BLIP、LLaVA、FLAVA、Qwen-VL 这些。

VLM 主要是处理语义信息，它能告诉你‘杯子在左边’，但不直接生成控制电机转动的底层指令。

也就是说它能看图片、理解文字、回答 “这是什么”“在哪里”“描述一下场景”等。但是不会控制机器人，不会动，不会导航，不会操作。

简单概括，VLM/VLN/VLA是三种类型的模型，分别能够完成一定范围内的AI相关的任务：
1） VLM是具身智能的 “感知大脑”；
2） VLN 是机器人的 “导航系统”；
3） VLA 是机器人的 “操作系统 + 四肢”。

1.1 核心定义与目标

1. VLN (视觉 - 语言 - 导航)

全称：Vision-Language Navigation
1. 核心目标：空间定位与路径规划。根据自然语言指令，在 3D 环境中自主找到目标位置并移动过去。
2. 本质：在 VLM 基础上，专门训练了导航能力。解决 “去哪里” 的问题。能够看环境、避障、规划路径、移动。只能移动，不能操作物体 （有脚没手）
3. 典型指令（能够执行）：理解 “去厨房”，“走到沙发旁”，“走到客厅的沙发旁边”
4. 不能执行：不会抓取、不会开门、不会倒水，不能操作物体。

2. VLA (视觉 - 语言 - 动作)

  1. 全称：Vision-Language-Action
  2. 核心能力：任务执行与物体操作。即要理解指令、感知环境，还要生成连续动作序列，与世界进行物理交互。
  3. 本质：解决 “做什么、怎么做” 的问题。即能移动，也能操作物体 （有脚有手）
  4. 典型指令：“把桌上的杯子拿起来，倒进碗里”，“打开抽屉，拿里面的扳手”，“去厨房，绕过桌子，打开冰箱”

1.2 关键区别对比

对比维度 VLN (视觉 - 语言 - 导航) VLA (视觉 - 语言 - 动作)
核心能力 空间推理、路径规划、避障移动 任务理解、手眼协调、物体操纵、序列执行
动作空间 离散或低维连续、移动为主 (前进、左转、右转) 连续、复杂 (抓取、旋转、按压、放置、力控)
输出结果 导航决策、移动轨迹点、路径序列 机器人关节角度、电机扭矩、末端执行器轨迹 (可直接驱动硬件)
任务复杂度 单一目标、长时序移动 (从 A 到 B) 多步骤、组合任务 (导航 + 识别 + 操作 + 反馈)
环境交互 低（主要是避障，不改变环境） 高（主动改变物体状态，强物理交互）
技术基础 SLAM、空间语义地图、强化学习 VLM (视觉 - 语言模型) + 动作生成、端到端策略网络
代表模型 VLN-Verse, VLN-R1, NavCoT Google RT-2, Meta VC-1, OpenVLA, SayCan
主要应用 导览机器人、仓储 AGV、四足巡检、家用导航 服务机器人、工业机械臂、灵巧手操作、自动驾驶

1.3 通俗理解：从 “导航” 到 “干活”

可以将机器人完成一个复杂任务（如 “去厨房拿杯水”）拆解为两个阶段：

1. VLN 阶段 (找路)

1.理解 “厨房”、“杯子” 的语义。

2.分析家居 3D 结构，规划从当前位置到厨房的路径。

3.实时视觉避障，移动到目标地点。

到此为止，是 VLN 的能力边界。

2. VLA 阶段 (操作)

1.到达厨房后，精准识别杯子的位置、姿态。

2.控制机械臂移动、张开手指、精准抓取（控制力度防碎）。

3.拿起杯子，移动到指定位置，平稳放下。

这一步，必须依靠 VLA 模型。

1.4 技术演进关系

粗略的，不严谨的说：
VLN 是 VLA 的基础子集：几乎所有 VLA 模型都内置了 VLN 的导航能力，因为执行复杂操作前通常需要先移动到目标位置。
VLA 是 VLN 的能力延伸：VLN 解决 “移动”，VLA 在 “移动” 之上增加了 “操作”，形成完整的 “感知 - 理解 - 行动” 闭环。

但这也不绝对，VLA 强调的是 Action（动作输出），VLN 强调的是 Navigation（移动规划）。两者是能力相互交叠关系，并不是严格的父子集关系。

在学术上，VLN 是一类任务，VLA 是另一类更通用的任务，不是集合包含关系。

在工业界，很多 VLA 模型是 “桌面级” (Table-top) 的。例如一个控制机械臂分拣零件的 VLA 模型，它具备极强的物理交互能力（Action），但它根本不需要“导航”（VLN）功能，因为它固定在桌子上。

1.5 总结

VLN：定位专家，擅长在大空间里找路、移动，核心是空间理解。
VLA：全能操作手，既能导航又能精准干活，核心是任务执行与物理交互。

2 「大脑」和「小脑」指什么？

大脑和小脑是从由人类的生理结构做类比的两类不同的任务，延伸到执行这两类任务的系统/模型。

大脑 – 思考做什么，策略规划；

小脑 – 思考怎么做，和最后进行动作执行。关节、伺服电机控制等，都属于小脑范畴。

2.1 大脑 = 高层推理、规划、决策系统

  1. 一句话：大脑负责 “想干什么”。也常被叫做：具身智能中枢，具身大模型 / 具身 Agent，任务规划器（Task Planner），高层认知模块
  2. 大脑负责：

• 理解自然语言指令

• 逻辑推理

• 任务拆解（把 “倒杯水” 拆成：去厨房→拿杯子→接水→拿回）

• 长期记忆、环境语义地图

• 做宏观决策：下一步该干什么

  1. 典型代表：

• LLM + 视觉语言模型（VLM）

• SayCan 中的语言规划器

• 具身智能的 “决策大脑” RT-2、RoboCat 高层部分

2.2 小脑 = 底层运动控制、执行、实时反馈系统

  1. 一句话：小脑负责 “具体怎么动”。也叫：运动控制器，策略网络（Policy Network），低层级控制模块，伺服控制、轨迹生成，小脑的核心特征是 高频 (High-frequency) 和 低延迟。
  2. 小脑负责：

• 实时生成机器人关节 / 轮子 / 云台的连续动作

• 避障

• 抓取、力控、精细操作

• 快速闭环反馈

• 做微观动作：怎么动、怎么抓、怎么走

  1. 典型代表：

• 端到端策略网络（Diffusion Policy、VIN、BC-Z）

• 传统机器人控制（MPC、PID、伺服驱动）

• VLA 模型里的动作输出头

2.3 大小脑和VLN/VLA的关系

  1. 四个概念各自定位：

• 大脑（LLM/VLM）理解任务 → 做规划 → 输出子目标例：“去厨房 → 找到杯子 → 抓取”

• 小脑（Policy / Control）执行子目标 → 输出关节 / 轮子动作

• VLN 模型 = 简化版的 “导航专用大脑 + 小脑” 打包体 ，它既有规划，又有移动控制

• VLA 模型 = 更通用的“通用大脑 + 小脑”一体化模型 ，从语言理解 → 视觉感知 → 直接输出机器人动作

说明：如果是一个端到端VLA模型（如 Google RT-2），它确实模糊了大小脑的界限。

但在很多架构中，VLA 仅作为 “技能大脑” ，它输出的是 6DoF 姿态或末端位移。真正的 “小脑”（底层 PID 控制、1kHz 的关节电流控制）通常还在传统的机器人控制器里，不在深度学习模型里。

所以VLA 通常是指 “高层动作策略”，而极底层的电机伺服控制目前仍多归属于“小脑”范畴。

  1. 一句话总结

• 大脑：负责 “想”、规划、任务理解。

• 小脑：负责 “动”、实时控制、执行。

• VLN：只会导航的完整小智能体。

• VLA：会干活 + 导航的通用智能体。

• VLN 和 VLA 都是 “完整小机器人”，不是大脑或小脑。

当然，如上面说的，一些非端到端的VLA是不含小脑部分的，主要做大脑。且业界现在流行和成熟的模式就是使用VLA+DiT双模型（大小脑分层）进行具身智能完整模型部署。

3 具身智能模型 指什么？

业界所属的具身智能模型，一般是指能让机器人 / 智能体看懂、听懂、会规划、会行动的整套 AI 模型体系，不是单一模型，而是多层级、多流派的组合。

3.1 具身智能模型：指什么、包含什么？

1. 定义（业界共识）

具身智能模型 = 能把感知（视觉 / 听觉）、语言理解、任务规划、物理动作控制 打通的 AI 系统，让机器在真实 / 虚拟世界里自主完成复杂任务（导航、抓取、操作、推理）。

2. 核心范围（完整技术栈）

 【感知层】VLM（视觉-语言模型）：看懂环境、听懂指令

 【导航层】VLN（视觉-语言-导航）：空间定位、路径规划、避障

 【动作层】VLA（视觉-语言-动作）：通用操作、抓取、工具使用

 【预测层】世界模型（World Model）：预判环境、物理规律、未来状态

 【控制层】小脑（运动策略）：实时电机控制、力控、稳定执行

包含了：VLM - 底座，VLN/VLA - 技能，世界模型 - 预判，小脑 - 执行。

3.2 三大主流流派（2026 年业界公认）

1. 大小脑分层架构（VLM + 规划 + 控制，仿生流派）

核心理念：大脑（VLM+LLM）负责思考规划，小脑负责运动执行。

传统大小脑：

大脑 = 独立任务规划器（输出子目标 / 语言指令 / 路点）

小脑 = 独立运动控制器（接收高层指令执行）

跟上面的端到端VLA的双系统不一样。双系统VLA的本质是：一个大模型拆成两个耦合的部分，而传统大小脑分层，是两个完全独立的系统，靠消息 / API 通信。

代表团队：

• 阿里达摩院（RynnBrain）

• 微软、北大、上交、华为具身团队

架构：

• 大脑：包含两个部分：

1）VLM+LLM → 理解、规划、记忆

2）VLN/VLA → 技能策略

• 小脑：运动控制器 → 实时执行

优势：

• 利用 VLM 预训练泛化

• 稳定、易调试、可解释

• 幻觉少、落地稳健

缺点：

• 模块多、集成复杂

• 端到端优化难度大

2. 端到端 VLA 流派（最火、工业落地主流）

核心理念：一个模型从视觉 + 语言直接输出动作，不拆分模块。

传统VLA将动作部分作为模型的最后一层，而采用专门的内置模型模块（DiT）来进行动作控制。

现代双系统 VLA = VLM backbone（大脑） + DiT 动作专家（小脑）；
    
VLM = 特征提取 Backbone（不是任务分解、不是符号规划）；
    
DiT = 模型内置的动作输出头（不是外挂控制模块）；
    
本质是：一个大模型拆成两个耦合的部分，输入指令 + 图像 → 输出动作 → 算损失 → 同时更新 VLM + DiT ，是一个完整端到端策略。

代表团队 / 公司：

• Figure AI、谷歌 Gemini Robotics、π（Physical Intelligence）、Covariant

• 国内：智元、自变量（Wall-E）、百度具身大模型

代表模型：

• RT-2、RT-1（谷歌）

• Fuyu-VLA、Pi0、Figure 01

• 国内：Wall-A、智元具身模型

优势：

• 端到端、简洁、易部署

• 语义与动作强对齐

• 工业场景落地最快

缺点：

• 泛化差、依赖海量真机数据

• 长任务易出错、难修正

3. 世界模型（World Model）流派（英伟达主推、下一代方向）

核心理念：先在脑中建模整个世界，再基于预测做决策。

代表团队：

• 英伟达（Jim Fan）：DreamZero、DreamDojo、LWM（大世界模型）

• DeepMind、Tesla Optimus、ACE Robotics（Kairos）

核心能力：

• 环境建模、物理规律学习

• 动作预判、“白日梦” 预演

• 跨场景强泛化

优势：

• 少数据、强泛化、抗干扰

• 长时序稳定、可反思优化

缺点：

• 计算量大、训练复杂

• 实时性弱、落地慢

3.3 传统VLA，双系统VLA和传统大小脑分层对比

维度 传统端到端单模型 VLA（老派） 现代双系统 VLA（VLM+DiT） 传统大小脑分层架构
流派归属 端到端 VLA 端到端 VLA（进化版） 大小脑分层（仿生）
模块关系 一体模型 耦合、可微、联合训练 完全解耦、独立训练
VLM 角色 整模型一体 特征 Backbone（认知） 独立规划器（决策）
动作模块 模型最后一层 DiT 动作专家（执行头） 独立控制模块（外挂）
梯度回传 全链路可微 全链路可微（VLM↔DiT） 不可微（规划→执行断梯度）
核心目标 输入→动作直接映射 输入→动作直接映射（更快更稳） 解耦规划与执行、安全可解释
代表 RT-2、早期 OpenVLA Figure Helix、GR00T、RT-4 SayCan、RynnBrain、传统机器人 + LLM

3.4 VLM / VLN / VLA / 世界模型：关系与定位

  1. 层级关系：VLM → VLN（增加导航）→ VLA（增加操作）→ VLA + 世界模型（增加预测）

• VLM：只会看和懂

• VLN：VLM + 导航（只会走）

• VLA：VLM + 导航 + 操作（会走会干活）

• 世界模型：给 VLA 装上 “预判能力”

  1. 在 “大脑 - 小脑” 体系中的位置

【大脑（认知）】

• LLM + VLM → 理解、语义、规划

• 世界模型 → 预测、物理推理

【技能层】

• VLN（导航专用）

• VLA（通用操作）

【小脑（执行）】

• 运动控制、实时执行

3.5 三大流派对比

流派 代表 核心思路 优势 短板 落地阶段
VLA 端到端 Figure、谷歌、Pi 一模型通吃：视觉→语言→动作 简洁、工业快落地 泛化差、数据饥渴 量产商用（2025-2026）
世界模型 英伟达、Tesla 先建模世界，再决策 强泛化、少数据、预判 复杂、实时弱、难训练 实验室→试点（2026 起）
大小脑分层 阿里、微软 大脑（VLM）+ 小脑（控制） 稳健、泛化好、易调试 模块多、集成难 商用 + 研发并行

3.6 一句话类比总结

• VLM = 眼睛 + 耳朵

• VLN = 会走路的 VLM

• VLA = 会干活的 VLN（当前主流）

• 世界模型 = 会预判未来的 VLA（下一代）

• 三大流派：分层架构最稳健 → VLA 快要落地 → 世界模型强泛化
