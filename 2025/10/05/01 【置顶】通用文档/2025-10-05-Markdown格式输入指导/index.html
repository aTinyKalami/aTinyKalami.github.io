02 Markdown格式输入指导

十二亚晖 2025-10-05 09:00:00
Categories： 通用 Tags： Markdown 语法 规则

MarkDown格式文档输入规则

Markdown 是一种轻量级标记语言，核心思想是用简单易读、易写的纯文本格式来编写文档，然后可自动转换成带有有效格式的 HTML 页面。

【参考指导】 详细的Markdown格式输入指导 → 可以点我。

1. 文档命名要求

按照Markdown格式撰写的Github所有博客文章都要存放在_posts 文件夹中。单击它，然后单击 创建文件按钮，即可创建一个新的文件。

系统要求文章按照下面规则命名：年年年年-月月-日日-文件名.md。

即文件名命名的格式：year-month-day-name.md，其中 year 是四位数，month 和 day 是两位数。“name” 可以自定义，扩展名 md 则用于 Markdown。

2. Markdown内容输入格式

使用Markdown格式包括文章标题、文本样式、代码块、超链接等基础语法：

2.1 标题

使用 # 字符来定义标题级别。提示： 为了美观，建议在 # 和标题文字之间加一个空格。

第一行内容以一个#号开始，然后接一个空格，然后是标题文字。这样就可以在markdown格式中生成一级标题。

使用#符号数量对应标题级别，最多支持六级标题（即#为一级标题，##为二级，###为三级，以此类推）。

例如：

一级标题

二级标题

三级标题

GitHub会自动根据标题生成目录，点击可快速跳转。

2.2 换行

在一行的末尾添加两个或多个空格，然后按回车键,即可创建一个换行

语法：

我是第一行， 我是第二行

效果：

我是第一行，
我是第二行

2.3 强调（粗体与斜体）

使用 _和* 字符来包裹文本，实现加粗和斜体效果。两个字符效果一样，使用数量不同效果不同。使用1个为斜体。2个为加粗体，3个为粗斜体。

斜体：前后各使用1个字符。

语法： *斜体文本* 或 _斜体文本_;
效果： 斜体文本

粗体：前后各使用2个字符。

语法： **粗体文本**或 __粗体文本__;
效果： 粗体文本

粗斜体：前后各使用3个字符。

语法： ***粗斜体文本*** 或 ___粗斜体文本___;
效果： 粗斜体文本

2.4 链接

1. 链接-行内式

使用 [] 和 () 字符。用于文中随时带入超链接，便于关联阅读。

语法： [链接文本]``(链接地址 +"可选的标题")

效果： 链接文本

如： [访问谷歌]``(https://www.google.com "搜索引擎")

例子： 访问谷歌

效果： 鼠标悬停显示搜索引擎，点击则访问谷歌URL链接

2. 链接-参考式

同样使用 [] 和 () 字符。用于文中多处使用同一个超链接。 在文章结尾统一给出。

这是一个[参考式链接] [1]的例子。

然后在文档任意位置（通常是末尾）定义：

[1]:（https://example.com “可选标题”）

如文章中文本：这里需要[访问谷歌][1]。

文档末尾定义：

[1]：https://www.google.com"搜索引擎“

(此处未验证，需要在文章中确认）

2.5 图片

语法与链接几乎完全相同，也是使用 [] 和 () 字符,只是在最前面加一个 !。

语法：![图片替代文本](图片链接地址 "可选的标题")

2.6 引用

要创建对一段文字块进行引用，要在文字段落前加上 > 符号。

引用显示效果是1）在文字段前有粗竖线分隔，2）文字字体为灰色。

语法：

>我是一段引用文字段落块

效果：

我是一段引用文字段落块

2.7 表格

管道符 | 是必须的，用于分隔列；分隔线 第二行必须包含，用于定义列对齐方式；空格 不影响渲染，主要用于提高可读性。

基本表格语法示例：

表头1 表头2 表头3
单元格1 单元格2 单元格3
单元格4 单元格5 单元格6

对齐方式：

:--- 左对齐

:---: 居中对齐

---: 右对齐

对齐表格示例：

左对齐 居中对齐 右对齐
内容1 内容2 内容3
内容4 内容5 内容6

2.8 Pre Code代码块

pre code 代码块，用来在页面中呈现独立代码块。

pre：预格式化文本，保持代码的缩进和换行

code：表示代码内容

class="language-xxx"：指定代码语言用于语法高亮

基础语法：

三个斜点开头加语言类型 换行写代码内容 三个斜点收尾
