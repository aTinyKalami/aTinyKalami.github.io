基础知识 - 大模型API接口调用参数说明

十二亚晖 2025-10-05 09:00:00
Categories： AI Tags： AI 技术 洞察

大模型的API调用，根据模型类别、用途功能，有很多不同的方面。

可以基于硅基流动的API文档进行完整和系统的学习与归档，参考硅基流动API文档。

业界当前大模型API调用，OpenAI基本形成事实标准，各个厂家新推出的模型的API调用，都遵循OpenAI的GPT的调用格式（Formate）。

OpenAI 的模型调用API参数总览

参数名 含义与作用 常见取值/类型 使用场景说明
model 指定要使用的模型 字符串，如 gpt-4o, gpt-3.5-turbo 必须参数，根据需求选择合适的模型版本
messages 对话消息列表 数组对象，包含 role 和 content 核心输入，支持多轮对话记忆
temperature 控制输出的随机性 0.0 到 2.0，默认 1.0 低值(0.1-0.5)：确定性强，适合代码生成、问答；高值(0.7-1.0)：创意性强，适合写作、构思
max_tokens 限制生成内容的最大长度 整数，如 100, 500, 1000 需考虑上下文窗口限制，防止生成截断
top_p 核采样，从概率质量前P%的token中抽样 0.0 到 1.0，默认 1.0 与temperature二选一使用。低值：确定性强，用于代码生成等严谨要求场景，高值：宽松，创意性强。
stop 停止序列 字符串或数组，如 "\n", ["。", "！"] 遇到指定序列时停止生成，节省token
stream 是否流式输出 布尔值，true/false true：实时显示，适合聊天应用；false：一次性返回
presence_penalty 存在惩罚，降低重复已出现词汇的可能性 -2.0 到 2.0，默认 0.0 正数减少重复，适合长文本生成
frequency_penalty 频率惩罚，降低重复频繁出现词汇的可能性 -2.0 到 2.0，默认 0.0 正数减少高频词重复，提升多样性
tools 定义模型可调用的函数/工具列表 数组对象 实现工具调用，扩展模型能力
tool_choice 控制函数/工具调用行为 字符串/对象，如 "auto", "none" auto：模型自行决定；none：不调用；指定函数：强制调用
logit_bias 对数偏置，修改指定token出现可能性 对象，如 {"15043": -100, "1234": 50} 精确控制特定词汇的生成概率

不同场景的创造性控制参数（temperature与Top_P）组合举例:

确定性强，适合事实问答：temperature=0.2, top_p=0.1

平衡创造性，通用场景：temperature=0.7, top_p=0.9

高创造性，适合创意写作：temperature=1.0, top_p=0.95

核心参数使用详细说明

temperature：

用途：这个参数用于控制模型输出的随机性或多样性。temperature 的值可以从 0 到 2 范围内调整。默认1。

行为：当 temperature 设置得较低（例如 0.1），模型的回答倾向于更加确定、一致和重复性高。当 temperature 设置得较高（例如 2），模型的回答则更加多样化和创造性。

典型用途：在需要确保回答的一致性和准确性时使用较低的 temperature，在探索不同可能性或创造性回答时使用较高的 temperature。

top_p：

用途：这个参数用于控制生成过程中采样的“词汇池”的大小，称为“nucleus sampling”。与temperature一般二选一。

行为：top_p 是一个概率阈值，只有累积概率达到这个阈值的最高概率词汇会被考虑进生成过程。例如，top_p 设置为 1 意味着没有限制（即排100%的生成内容都可以被选中），模型会从整个词汇库中选择词汇。设置为 0.1 表示只从累积概率达到前 10% 的词汇中选择。

典型用途：用于增加输出的多样性而不是严格地基于最高概率的词汇选择。

stream：

用途：这个参数决定了 API 的响应方式，是以流的形式还是一次性返回完整的响应。

行为：当 stream 设置为 True 时，API 会逐渐返回生成的内容，即边生成边返回。当设置为 False 时，API 会等到整个生成过程完成后才返回结果。

典型用途：在需要实时接收生成数据时（如实时显示聊天回复），可以设置为 True。在大多数情况下，特别是需要一次性获取完整回答的情况下，设置为 False 是更常见的选择。

Messages：

用途：这个参数用于给大模型提供每轮输入的Tokens，从而启动大模型的Prefill环节。如果要让大模型有记忆，就要在程序中保留历史的输入和输出，并且每次在调用中作为输入通过Messages API的参数提供给模型。
