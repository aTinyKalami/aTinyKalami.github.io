一. 安装Ubuntu Linux（必选，基础环境配置）

1.安装Ubuntu Linux系统

Linux是动手实操的基础环境要求，安装Linux的方法有几种，比如直接在本机安装Linux操作系统，或者本机Window操作系统中通过WSL (Windows Subsystem Linux)安装Linux，或者通过安装VMware虚拟机环境，再通过虚机管理工具安装Linux等。

我实际使用的是通过WSL，在本机Windows环境中安装Linux。后续有时间再尝试其他方式。

在本机通过WSL安装Ubuntu，构建镜像多实例的参考视频 → 点我

我本机 Windows 版本太旧，不支持新版 WSL (Windows Subsystem Linux)命令。使用手动在微软软件商店中下载和安装Ubuntu：

1）打开 Microsoft Store
2）搜索 “Ubuntu”
3）选择 “Ubuntu 20.04 LTS” 或 “Ubuntu 22.04 LTS” 安装
4）安装完成后从开始菜单启动 Ubuntu，会提示设置账号（全小写）和密码

2.2 部署/恢复Linux镜像

WSL支持基于一个Linux镜像部署多个实例，并且通过命令窗口在多个实例环境中切换，从而避免Linux环境中的各种程序的各种依赖冲突。

基于.tar文件在WSL中部署一个Linux实例的命令是： wsl –import。

这个命令可以将 .tar 备份文件作为全新的、独立的 Linux 发行版导入到您的 Windows 的wsl系统中。

需要在 Windows 命令行 (PowerShell 或 CMD) 中执行此命令（使用时需去掉<>符号），地址路径描述要加双引号” “：

WSL – import命令的参数要求详细介绍：

参数项 是否必选 描述
<发行版/新实例名称> 必选 你想给新实例起的名字（例如 Ubuntu-Work）。之后可通过 wsl -d <名称> 启动。
<安装路径> 必选 WSL 虚拟硬盘文件（ext4.vhdx）存放的目标文件夹。建议在执行命令前先手动创建该目录。
<源文件路径> 必选 之前导出的 .tar 备份文件或下载的 rootfs 压缩包的完整路径。
--version <1/2> 可选 指定导入为 WSL 1 还是 WSL 2。如果不指定，将使用系统当前的全局默认版本。
--vhd 可选 如果源文件本身就是 .vhdx 虚拟硬盘文件（而非 .tar 压缩包），则需带上此参数（仅限 WSL 2）。

3. Linux环境实例操作基本命令

Linux环境的操作相关的命令分为两类：WSL 管理命令（在 Windows 命令行中执行）和 Linux 基本操作命令（在 Ubuntu 环境中执行）。

3.1 WSL 环境管理命令 (在 Windows CMD命令行窗口中执行)

这些命令用于管理您的整个 Linux 实例（如 Ubuntu-20.04 或 Evalscope_Test）。

目标 命令 备注
列出所有实例 wsl –list –all 或 wsl -l -v “查看所有已安装实例及其状态 (Running, Stopped)。”
启动/进入环境 wsl -d 启动并进入指定的实例。 (如: wsl -d Evalscope_Test)
进入默认环境 wsl 如果不指定名称，则进入默认设置的实例
终止环境 wsl –terminate 或 wsl -t 强制关闭/停止正在运行的实例。 (兼容旧版 WSL)
设置默认值 wsl –set-default 将指定的实例设为默认启动项
备份环境 wsl –export <File.tar> 将实例导出为备份文件
部署环境 wsl –import <File.tar> 从备份文件导入为新实例
删除环境 wsl –unregister 警告： 彻底删除实例及其所有数据，不可恢复。

二. 使用Windows Terminal（终端）工具

在 Windows 环境下，要操作WSL里的Ubuntu实例，需要在window的Power Shell（用Win +X激活，选择Power Shell）命令窗口中操作。PowerShell 不仅仅是启动 WSL 的“入口”，它更是管理、配置和跨系统协作的核心工具。

但是实际操作中经常需要在Powershell、CMD、Gitbash等多种命令行工具中切换，可以使用由微软官方提供的命令行系统工具Windows Terminal（中文名称：终端），在Windows操作系统自带的软件商店中搜索名称就可以找到。

Windows Terminal 是微软开发的现代终端工具，它最大的优势是能将 PowerShell、CMD、WSL (Linux) 以及 Git Bash 等全部集成在一个窗口中，并支持多标签和高度自定义。这是一个简单的多任务/实例管理工具，可以通过多窗口切换的方式同时操作多个环境，比如我本机管理员级的Powershell、CMD、Gitbash（需配置）和激活的各个不同的Linux镜像，如Evalscope等。

  1. 下载安装：打开 Windows 自带的微软商店，搜索 “Windows Terminal”（或“终端”），点击获取即可。这种方式支持后台自动更新。

  2. 使用方法：

  1. 快速启动与多标签
  • 新建标签： 点击顶部栏的 + 号（默认打开 PowerShell）。
  • 切换环境： 点击 + 号旁边的 下拉箭头，你可以直接选择进入 Powershell、CMD 、Git Bash（配置激活后）或激活的Ubuntu系统。
  • 快捷键：
  • Ctrl + Shift + T: 新建标签页。
  • Ctrl + Shift + W: 关闭当前标签页。
  • Ctrl + ,: 打开设置界面。
  1. 垂直/水平分屏

这是 Terminal 的神技，让你在一个窗口看多个屏幕：

  • 自动分屏： Alt + Shift + D
  • 水平分屏： Alt + Shift + -
  • 垂直分屏： Alt + Shift + + (实际上是 + 号那个键)
  1. 将 Git Bash 等其他命令行工具通过添加配置文件的方式添加到列表

默认情况下，Windows Terminal 可能不会自动识别 Git Bash。你可以手动添加它：

  1. 按 Ctrl + , 进入设置。
  2. 点击左下角的 “打开 JSON 文件”（或者在 UI 界面点击“添加新配置文件”）。
