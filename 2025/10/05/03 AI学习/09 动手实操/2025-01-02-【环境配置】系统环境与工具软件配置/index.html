部署/恢复Linux镜像

WSL支持基于一个Linux镜像部署多个实例，并且通过命令窗口在多个实例环境中切换，从而避免Linux环境中的各种程序的各种依赖冲突。

基于.tar文件在WSL中部署一个Linux实例的命令是： wsl –import。这个命令可以将 .tar 备份文件作为全新的、独立的 Linux 发行版导入到您的 Windows 系统中。

需要在 Windows 命令行 (PowerShell 或 CMD) 中执行此命令。

PowerShellwsl –import

参数 描述 示例值
您想给新实例起的名字。 Evalscope_Test
新实例的文件将要存储的目录。这个目录必须存在，且不应是系统默认的 AppData 路径。 D:\WSL_Environments\Test_Instance
导出的 .tar 文件的完整路径。 D:\WSL_Backups\ubuntu_evalscope_CLEAN.tar

通过对新实例用不同的名字进行区分，就可以在系统中拉起多个Linux环境实例。

Linux环境实例操作基本命令

相关的命令分为两类：WSL 管理命令（在 Windows 命令行中执行）和 Linux 基本操作命令（在 Ubuntu 环境中执行）。

1. WSL 环境管理命令 (在 Windows 命令行执行)

这些命令用于管理您的整个 Linux 实例（如 Ubuntu-20.04 或 Evalscope_Test）。

目标 命令 备注
列出所有实例 wsl –list –all 或 wsl -l -v “查看所有已安装实例及其状态 (Running, Stopped)。”
启动/进入环境 wsl -d 启动并进入指定的实例。 (如: wsl -d Evalscope_Test)
进入默认环境 wsl 如果不指定名称，则进入默认设置的实例
终止环境 wsl –terminate 或 wsl -t 强制关闭/停止正在运行的实例。 (兼容旧版 WSL)
设置默认值 wsl –set-default 将指定的实例设为默认启动项
备份环境 wsl –export <File.tar> 将实例导出为备份文件
部署环境 wsl –import <File.tar> 从备份文件导入为新实例
删除环境 wsl –unregister 警告： 彻底删除实例及其所有数据，不可恢复。

2. 文件与目录操作

这些是在您进入 Ubuntu 环境后，进行日常操作、管理文件和退出虚拟环境时使用的命令。

  1. 虚拟环境管理 (Venv)
目标 命令 备注
创建虚拟环境 python3 -m venv 在当前目录下创建虚拟环境。
进入环境 source /bin/activate 激活虚拟环境。 提示符前会显示 ()。
退出环境 deactivate 退出当前的虚拟环境，回到系统 Base 环境。
导出依赖列表 pip freeze > requirements.txt 备份当前环境的所有 Python 依赖。

2.文件与目录操作

目标 命令 备注
列出文件 ls 列出当前目录下的文件和文件夹。 (ls -l 显示详细信息)
进入目录 cd 切换到指定目录。 (cd .. 返回上一级)
创建目录 mkdir 创建新目录。
创建文件 touch 创建一个空文件。
复制文件 cp 复制文件或目录。 (-r 复制目录)
移动/重命名 mv 移动文件或重命名文件。
删除文件 rm 删除文件。
删除目录 rm -rf 警告： 强制递归删除目录及其所有内容，不可恢复。
查看文件内容 cat 打印文件所有内容。
  1. 系统信息与退出
    目标 命令 备注

    查看系统信息 uname -a 查看 Linux 内核信息。
    退出 Ubuntu exit 退出当前的 Bash Shell，通常也会终止 WSL 实例运行 (旧版 WSL)。
