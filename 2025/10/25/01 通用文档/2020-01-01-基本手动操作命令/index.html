01 Hexo框架的Github文章手工部署常用命令

十二亚晖 2025-10-25 09:00:00
Categories： 通用 Tags： Hexo，博客，操作

1.【本地→云端 文件同步加页面发布】

当在自己的本地电脑写了新文章以后，要同步到云端，并且发布:

1.进入Hexo项目文件夹:

cd /d/handson/github/blog #切换目录到项目文件夹

2.生成静态页面并通过Hexo部署网站（即发布到 .io 仓库）

hexo clean && hexo generate && hexo deploy

3.推送源码文件夹到 blog-code-folder 仓库，包括同步文章、配置变更等到云端:

git add . #添加新文件
git commit -m "本地内容变更" #提交更改
git push origin main #推到远端代码仓

【本地 → 云端】命令合成一行代码，打开git直接输入：

cd /d/handson/github/blog&&git status&&git add .&&git commit -m “本地内容变更”&& git push origin main&&hexo clean && hexo generate&&hexo deploy

2.【云端→本地 文件同步加页面发布】

如果我在Github网页侧更新代码托管仓中的Source文件夹的文章内容，要同步到本地并且发布：

1.进入Hexo项目文件夹:

cd /d/handson/github/blog #切换目录到项目文件夹

2.拉取远程最新更改到本地:

git pull origin main #拉取云端最新文件到本地

3.生成静态页面并通过Hexo部署网站（即发布到 .io 仓库）

hexo clean && hexo generate && hexo deploy

【云端→本地】命令合成一行代码，打开git直接输入：

cd /d/handson/github/blog &&git pull origin main && hexo clean && hexo generate && hexo deploy

3.【如果云端和本地都修改了，就要同时拉取和推送】

3.1. 先拉取云端修改（可能会产生冲突）

git pull origin main

3.2. 如果有冲突，手动解决冲突

3.3. 然后推送本地修改

git push origin main

(本文完)
